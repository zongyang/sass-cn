# 等值运算

兼容性：Dart Sass ✓ |LibSass ✗|Ruby Sass 4.0.0(unreleased)

LibSass和旧版本的Ruby Sass认为没有单元的数字与带任意单位的数字相等。这种行为已经被弃用，并且在最近的版本中被删除了，因为它违反了传递性。

等值运算符返回两个值是否相同。<expression> == <expression>，返回两个表达式是否相等；<expression> != <expression>，返回两个表达式是否不相等。如果两个值是相同的类型和相同的值，则认为它们是相等的，这意味着不同的类型有不同的含义：

  • Numbers，如果具有相同的值和相同的单位，或者当它们的单位相互转换时，它们的值是相等的，那么它们就是相等的。
  • Srings，相同内容的不带引用的字符串和带引用的字符串被认为是相等的。
  • Colors，如果它们具有相同的红、绿、蓝和alpha值，则视为是相等的。
  • Lists，如果列表的内容是相同的，则列表也是相等的。逗号分隔的列表不等于空格分隔的列表，带括号的列表也不等于不带括号的列表。
  • Maps，如果键和值都相等，就是相等的。
  • true、false和null只等于它们自己。
  • Functions。函数是通过引用进行比较的，因此即使两个函数具有相同的名称和定义，如果不在相同的位置定义，它们也会被认为是不同的。
@debug 1px == 1px; // true
@debug 1px != 1em; // true
@debug 1 != 1px; // true
@debug 96px == 1in; // true

@debug "Helvetica" == Helvetica; // true
@debug "Helvetica" != "Arial"; // true

@debug hsl(34, 35%, 92.1%) == #f2ece4; // true
@debug rgba(179, 115, 153, 0.5) != rgba(179, 115, 153, 0.8); // true

@debug (5px 7px 10px) == (5px 7px 10px); // true
@debug (5px 7px 10px) != (10px 14px 20px); // true
@debug (5px 7px 10px) != (5px, 7px, 10px); // true
@debug (5px 7px 10px) != [5px 7px 10px]; // true

$theme: ("venus": #998099, "nebula": #d2e1dd);
@debug $theme == ("venus": #998099, "nebula": #d2e1dd); // true
@debug $theme != ("venus": #998099, "iron": #dadbdf); // true

@debug true == true; // true
@debug true != false; // true
@debug null != false; // true

@debug get-function("rgba") == get-function("rgba"); // true
@debug get-function("rgba") != get-function("hsla"); // true
