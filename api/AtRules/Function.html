# @function

函数允许您定义对SassScript值的复杂操作，并且可以在整个样式表中复用用这些函数。函数可以很方便地、易读的方式抽象出常见的公式和行为。

函数使用@function定义，这样书写@function <name>(<arguments…>){...}。函数的名称可以是任何Sass标识符。它只能包含通用语句，以及@return ，作为函数的返回值。函数使用普通的CSS函数语法调用。

@function pow($base, $exponent) {
  $result: 1;
  @for $_ from 1 through $exponent {
    $result: $result * $base;
  }
  @return $result;
}

.sidebar {
  float: left;
  margin-left: pow(4, 3) * 1px;
}

@function pow($base, $exponent)
  $result: 1
  @for $_ from 1 through $exponent
    $result: $result * $base

  @return $result

.sidebar
  float: left
  margin-left: pow(4, 3) * 1px

.sidebar {
  float: left;
  margin-left: 64px;
}

函数名与所有Sass标识符一样，将连字符和下划线视为相同的。这意味着scale-color和scale_color都指向同一个函数。这是Sass早期的历史遗留问题，当时它只允许在标识符名称中使用下划线。一旦Sass添加了对连字符的支持来匹配CSS的语法，这两种方法就等价了，从而使迁移更加容易。

虽然函数在技术上有可能产生诸如设置全局变量之类的副作用，但强烈建议不要这样做。使用mixin来解决这些问题，并使用函数来计算值。

# 参数

参数允许每次调用函数时定制它们的行为。参数是在函数名后面的@function中指定的，它是一个由圆括号包围的变量名列表。必须使用相同数量的参数（以SassScript表达式的形式）调用该函数。这些表达式的值可以作为相应的变量在函数体中使用。

参数列表后面也可以有逗号！这可以使重构样式表时更容易避免语法错误。

# 可选参数

通常，函数声明的每个参数都必须在包含该函数时传递。但是，您可以通过定义一个默认值来使参数成为可选的，如果没有传递该参数，则将使用该默认值。默认值使用与变量声明相同的语法：变量名后跟冒号和SassScript表达式。这使得定义灵活的函数API变得很容易，这些API可以以简单或复杂的方式使用。

@function invert($color, $amount: 100%) {
  $inverse: change-color($color, $hue: hue($color) + 180);
  @return mix($inverse, $color, $amount);
}

$primary-color: #036;
.header {
  background-color: invert($primary-color, 80%);
}

@function invert($color, $amount: 100%)
  $inverse: change-color($color, $hue: hue($color) + 180)
  @return mix($inverse, $color, $amount)


$primary-color: #036
.header
  background-color: invert($primary-color, 80%)

.header {
  background-color: #523314;
}

默认值可以是任何SassScript表达式，它们甚至可以引用前面的参数！

# 键名参数

当调用一个函数时，除了按参数列表中的位置传递参数外，还可以按名称传递参数。这对于具有多个可选参数的函数，或具有布尔参数的函数尤其有用。键名参数使用与变量声明和可选参数相同的语法。

$primary-color: #036;
.banner {
  background-color: $primary-color;
  color: scale-color($primary-color, $lightness: +40%);
}

$primary-color: #036
.banner
  background-color: $primary-color
  color: scale-color($primary-color, $lightness: +40%)

.banner {
  background-color: #036;
  color: #0a85ff;
}

因为任何参数都可以通过名称传递，所以在重命名函数的参数时要小心...这可能会使用户崩溃！将旧名称作为可选参数保留一段时间，并在有人传递时打印警告，这样他们就知道要迁移到新参数，这是很有帮助的。

# 不定参数

有时函数能够接受任意数量的参数是很有用的。如果@function声明中的最后一个参数以...结尾，然后将该函数的所有额外参数作为列表传递给该参数。这个参数称为参数列表。

@function sum($numbers...) {
  $sum: 0;
  @each $number in $numbers {
    $sum: $sum + $number;
  }
  @return $sum;
}

.micro {
  width: sum(50px, 30px, 100px);
}

@function sum($numbers...)
  $sum: 0
  @each $number in $numbers
    $sum: $sum + $number

  @return $sum

.micro
  width: sum(50px, 30px, 100px)

.micro {
  width: 180px;
}

# 不定键名参数

参数列表还可以用于接受不定键名参数。keywords接受一个参数列表，并返回这些额外的关键字参数的map，其中键名不包含$，传递给函数。

如果您没有将参数列表传递给keywords()，则该参数列表将不允许额外的关键字参数。这可以帮助函数的调用者确保他们的参数名称中没有出现任何拼写错误。

# 传递不定参数

就像参数列表允许函数接受任意位置或关键字参数一样，可以使用相同的语法将位置和关键字参数传递给函数。如果你传递一个后面跟着...作为函数调用的最后一个参数，它的元素将被视为附加的位置参数。类似地，map后面跟着...将被视为附加的关键字参数。你甚至可以一次把它们否传递过去！

$widths: 50px, 30px, 100px;
.micro {
  width: min($widths...);
}

$widths: 50px, 30px, 100px
.micro
  width: min($widths...)

.micro {
  width: 30px;
}

因为参数列表同时跟踪位置参数和关键字参数，所以可以使用它同时将这两个参数传递给另一个函数。这使得为函数定义别名变得超级容易！

@function fg($args...) {
  @warn "The fg() function is deprecated. Call foreground() instead.";
  @return foreground($args...);
}

@function fg($args...)
  @warn "The fg() function is deprecated. Call foreground() instead."
  @return foreground($args...)

# @return

@return表示调用函数后返回的计算结果。它只允许在@function主体中使用，并且每个@function必须以@return结束。

当遇到@return时，它立即结束函数并返回结果。尽早返回对于处理边缘情况非常有用。

@function str-insert($string, $insert, $index) {
  // Avoid making new strings if we don't need to.
  @if str-length($string) == 0 {
    @return $insert;
  }

  $before: str-slice($string, 0, $index);
  $after: str-slice($string, $index);
  @return $before + $insert + $after;
}

@function str-insert($string, $insert, $index)
  // Avoid making new strings if we don't need to.
  @if str-length($string) == 0
    @return $insert


  $before: str-slice($string, 0, $index)
  $after: str-slice($string, $index)
  @return $before + $insert + $after

# 其它函数

除了用户定义的函数之外，Sass还提供了一个内置函数的核心库，这些函数总是可用的。Sass的各种实现还使得在宿主语言中定义自定义函数成为可能。当然，您总是可以调用原生CSS函数（即使语法很奇怪）。
