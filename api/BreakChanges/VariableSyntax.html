# CSS变量语法

旧版本的LibSass和Ruby Sass与其他任何属性声明一样解析自定义属性声明，允许将所有SassScript表达式作为值。但这与CSS不兼容。

兼容性：Dart Sass ✓ |LibSass 3.5.0+|Ruby Sass 3.5.0+

CSS规范允许在自定义属性声明中使用几乎任何字符串。尽管这些值可能对任何CSS属性都没有意义，但是可以通过JavaScript访问它们。当它们被解析为SassScript值时，原本是有效的普通CSS语法无法解析。例如，Polymer library使用它来支持原生CSS混合器：

:root {
    --flex-theme: {
        border: 1px solid var(--theme-dark-blue);
        font-family: var(--theme-font-family);
        padding: var(--theme-wide-padding);
        background-color: var(--theme-light-blue);
    };
}

:root {
    --flex-theme: {
        border: 1px solid var(--theme-dark-blue);
        font-family: var(--theme-font-family);
        padding: var(--theme-wide-padding);
        background-color: var(--theme-light-blue);
    };
}

为了提供与原生CSS的最大兼容性，最新版本的Sass要求在插值表达式中写入自定义属性值中的SassScript表达式。插值表达式也适用于较老的Sass版本，因此推荐用于所有样式表。

$accent-color: #fbbc04;

:root {
    // WRONG, will not work in recent Sass versions.
    --accent-color-wrong: $accent-color;

    // RIGHT, will work in all Sass versions.
    --accent-color-right: #{$accent-color};
}

$accent-color: #fbbc04

:root
    // WRONG, will not work in recent Sass versions.
    --accent-color-wrong: $accent-color

    // RIGHT, will work in all Sass versions.
    --accent-color-right: #{$accent-color}

:root {
    --accent-color-wrong: $accent-color;
    --accent-color-right: #fbbc04;
}

因为插值表达式会从带引号的字符串中删除引号，所以可能有必要将它们封装在meta.inspect()函数中，以保留它们的引号。

@use "sass:meta";

$font-family-monospace: Menlo, Consolas, "Courier New", monospace;

:root {
    --font-family-monospace: #{meta.inspect($font-family-monospace)};
}

@use "sass:meta"

$font-family-monospace: Menlo, Consolas, "Courier New", monospace

:root
    --font-family-monospace: #{meta.inspect($font-family-monospace)}

:root {
    --font-family-monospace: Menlo, Consolas, "Courier New", monospace;
}
