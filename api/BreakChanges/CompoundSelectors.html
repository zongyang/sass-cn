# 扩展复合选择器

LibSass目前允许扩展像.message.info这样的复合选择器，但是扩展的方式与@extend的工作方式不匹配。

兼容性：Dart Sass ✓ |LibSass ✗|Ruby Sass ✗

当一个选择器扩展另一个选择器时，Sass样式所有匹配扩展器的元素，就好像它们也匹配被扩展的类一样。换句话说，如果您编写.heads-up {@extend .info}，它的工作原理就像您将HTML中的class="heads-up"替换为class="heads-up info"一样。

按照这个逻辑，您应该期望heads-up {@extend .message.info}能够像class="heads-up"替换class="heads-up info message"那样工作。但是现在在LibSass和Ruby它不向每个包含.info或.message的选择器添加.heads-up，而只将它添加到包含.info.message的选择器中。

// These should both be extended, but they won't be.
.message {
    border: 1px solid black;
}
.info {
    font-size: 1.5rem;
}

.heads-up {
    @extend .message.info;
}

// These should both be extended, but they won't be.
.message
    border: 1px solid black

.info
    font-size: 1.5rem

.heads-up
    @extend .message.info

为了解决这个问题，避免更多的混乱，并保持实现的干净和高效，扩展复合选择器的能力在Dart Sass中不受支持，将在LibSass的未来版本中删除。为了兼容性，用户应该分别扩展每个简单选择器：

.message {
    border: 1px solid black;
}
.info {
    font-size: 1.5rem;
}

.heads-up {
    @extend .message, .info;
}

.message
    border: 1px solid black

.info
    font-size: 1.5rem

.heads-up
    @extend .message, .info


.message, .heads-up {
    border: 1px solid black;
}

.info, .heads-up {
    font-size: 1.5rem;
}

因为Sass不知道HTML将会使用哪些样式，所以任何@extend可能都需要生成额外的选择器，而这些选择器并不特别适用于您的HTML。当从扩展复合选择器切换时尤其如此。

大多数情况下，这些额外的选择器不会造成任何问题，只会向压缩的CSS添加几个额外的字节。但是一些样式表可能更依赖于旧的行为。在这种情况下，我们建议用占位符选择器替换复合选择器。

// Instead of just `.message.info`.
%message-info, .message.info {
    border: 1px solid black;
    font-size: 1.5rem;
}

.heads-up {
    // Instead of `.message.info`.
    @extend %message-info;
}

// Instead of just `.message.info`.
%message-info, .message.info
    border: 1px solid black
    font-size: 1.5rem

.heads-up
    // Instead of `.message.info`.
    @extend %message-info

.heads-up, .message.info {
    border: 1px solid black;
    font-size: 1.5rem;
}
