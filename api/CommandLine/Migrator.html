# 迁移器

Sass迁移器会自动更新您的Sass文件，以帮助您迁移到该语言的最新和最好的版本。它的每个命令都迁移一个单独的特性，以便尽可能多地控制更新的内容和更新的时间。

# 使用方法

告诉Sass迁移器您想要运行的迁移和您想要迁移的Sass文件：

sass-migrator <migration> <entrypoint.scss...>

默认情况下，迁移器将只更改您在命令行上显式传递的文件。传递--migrate-deps选项告诉迁移器要更改使用@use规则、@forward规则或@import规则加载的所有样式表。如果您想做一个测试运行，看看在不实际保存更改的情况下会做哪些更改，那么您可以传递--dry-run --verbose（或者缩写-nv）选项。

$ cat style.scss
$body-bg: #000;
$body-color: #111;

@import "bootstrap";

@include media-breakpoint-up(sm) {
  .navbar {
    display: block;
  }
}
$ sass-migrator --migrate-deps module style.scss
$ cat style.scss
@use "bootstrap" with (
  $body-bg: #000,
  $body-color: #111
);

@include bootstrap.media-breakpoint-up(sm) {
  .navbar {
    display: block;
  }
}

# 安装

在可以安装Dart Sass的地方基本上都可以安装迁移器。

# 独立安装

Windows、Mac或Linux都可以安装Sass迁移器，从GitHub下载对应系统的安装包，并配置好环境变量。

# npm

如果您使用的是Node.js，可以通过npm安装Sass迁移器。

npm install -g sass-migrator

# Chocolatey

如果您使用Windows系统下的Chocolatey包管理器，可以通过以下命令来安装Sass迁移器。

choco install sass-migrator

# Homebrew

如果您使用Mac OS X下的Homebrew包管理器，可以通过以下命令来安装Dart Sass。

brew install sass/sass/migrator

# 全局选项

所有迁移器都可以使用这些全局选项。

# --migrate-deps

此选项（缩写为-d）告诉迁移器不仅要更改在命令行上显式传递的样式表，还要更改依赖于使用@use规则、@forward规则或@import规则的样式表。

$ sass-migrator module --verbose style.scss
Migrating style.scss
$ sass-migrator module --verbose --migrate-deps style.scss
Migrating style.scss
Migrating _theme.scss
Migrating _fonts.scss

模块迁移器假设任何依赖于使用@use规则或@forward规则的样式表都已经迁移到模块系统，因此即使传递了--migrate-deps选项，它也不会尝试迁移它们。

# --load-path

这个选项（缩写为-I）告诉迁移器一个加载路径，它应该在其中查找样式表。它可以被多次传递以提供多个加载路径。较早的加载路径将优先于较晚的加载路径。

从加载路径加载的依赖项被假定为第三方库，因此即使传递了--migrate-deps选项，迁移器也不会迁移它们。

# --dry-run

此标志（缩写为-n）告诉迁移器不要将任何更改保存到磁盘。相反，它会打印将要更改的文件列表。这通常--verbose选项一起使用，以打印本应该进行的更改的内容。

$ sass-migrator module --dry-run --migrate-deps style.scss
Dry run. Logging migrated files instead of overwriting...

style.scss
_theme.scss
_fonts.scss
_grid.scss

# --no-unicode

此标志告诉Sass迁移器仅将ASCII字符作为错误消息的一部分发送到终端。默认情况下，或者如果传递了--unicode，迁移器将为这些消息发出非ASCII字符。此标记不影响CSS输出。

$ sass-migrator --no-unicode module style.scss
line 1, column 9 of style.scss: Error: Could not find Sass file at 'typography'.
  ,
1 | @import "typography";
  |         ^^^^^^^^^^^^
  '
Migration failed!
$ sass-migrator --unicode module style.scss
line 1, column 9 of style.scss: Error: Could not find Sass file at 'typography'.
  ╷
1 │ @import "typography";
  │         ^^^^^^^^^^^^
  ╵
Migration failed!

# --verbose

这个标志（缩写为-v）告诉迁移器将额外的信息打印到控制台。默认情况下，它只打印已更改文件的名称，但是当与--dry-run选项结合使用时，它还打印这些文件更改的内容。

$ sass-migrator module --verbose --dry-run style.scss
Dry run. Logging migrated files instead of overwriting...
<==> style.scss
@use "bootstrap" with (
  $body-bg: #000,
  $body-color: #111
);

@include bootstrap.media-breakpoint-up(sm) {
  .navbar {
    display: block;
  }
}
$ sass-migrator module --verbose style.scss
Migrating style.scss

# 迁移

迁移器目前只支持一种迁移，但是随着Sass语言的不断发展，还会支持更多迁移！

# 模块

此迁移将旧的@import规则加载依赖项转换为通过@use规则加载。它不会天真地将@import更改为@uses-它会智能地更新样式表，使它们保持与以前相同的工作方式，包括：

  • 将命名空间加到其它使用到了的模块成员（变量、混合器和函数）。
  • 向使用成员的样式表添加新的@use规则，但不导入它们。
  • 将覆盖的默认变量转换为with子句。
  • 在其它使用到的成员文件中自动删除-和_前缀（因为否则它们将被认为是私有的，只能在声明它们的模块中使用）。
  • 将嵌套导入转换为使用meta.load-css() mixin。
$ cat style.scss
$body-bg: #000;
$body-color: #111;

@import "bootstrap";

@include media-breakpoint-up(sm) {
  .navbar {
    display: block;
  }
}
$ sass-migrator --migrate-deps module style.scss
$ cat style.scss
@use "bootstrap" with (
  $body-bg: #000,
  $body-color: #111
);

@include bootstrap.media-breakpoint-up(sm) {
  .navbar {
    display: block;
  }
}

因为模块迁移器可能需要修改成员定义和成员名称，所以使用--migrate-deps或者确保将包或应用程序中的所有样式表传递给它是很重要的。

# 加载依赖

即使没有传递--migrate-deps选项，模块迁移器也需要能够读取它所迁移的模块所依赖的所有样式表。如果迁移器未能找到依赖项，您将得到一个错误。

$ ls .
style.scss  node_modules
$ sass-migrator module style.scss
Error: Could not find Sass file at 'dependency'.
  ,
1 | @import "dependency";
  |         ^^^^^^^^^^^^
  '
  style.scss 1:9  root stylesheet
Migration failed!
$ sass-migrator --load-path node_modules module style.scss

如果在编译样式表时使用加载路径，请确保使用--load-path选项将其传递给迁移器。

不幸的是，迁移器不支持自定义导入器，但是它有内置的支持，可以通过在node_modules中搜索来解析以~开头的url，这与Webpack所支持的类似。
