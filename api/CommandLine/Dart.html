# Dart Sass命令行界面

# 使用方法

Dart Sass可执行文件可以在两种模式中调用。

# 一对一模式

sass <input.scss> [output.css]

一对一模式将单个输入文件（input.scss）编译为单个输出位置（output.css)。如果没有传递输出位置，则将编译后的CSS打印到终端。

如果扩展名为.SCSS，则将输入文件解析为SCSS；如果扩展名为.sass，则解析为缩进语法；如果扩展名为.CSS，则解析为原生CSS。如果它没有这三种扩展中的一种，或者它来自标准输入，则默认将其解析为SCSS。这可以用--indented标志来控制。

特殊字符串—可以用来代替输入文件，以告诉Sass从标准输入中读取输入。Sass默认将其解析为SCSS，除非传递了--indented标志。

# 多对多模式

兼容性 Dart Sass 1.4.0+

sass [<input.css>:<output.css>] [<input/>:<output/>]...

多对多模式将一个或多个输入文件编译为一个或多个输出文件。输入和输出用冒号分开。它还可以将一个目录中的所有Sass文件编译为另一个目录中具有相同名称的CSS文件。

​# Compiles style.scss to style.css.
$ sass style.scss:style.css

​# Compiles light.scss and dark.scss to light.css and dark.css.
$ sass light.scss:light.css dark.scss:dark.css

​# Compiles all Sass files in themes/ to CSS files in public/css/.
$ sass themes:public/css

在编译整个目录时，Sass将忽略名称以_开头的片段文件。您可使用片段文件来避免生成一堆不必要的输出文件。

# 配置选项

# 输入和输出

这些选项控制Sass如何加载输入文件以及如何生成输出文件。

# --stdin

此标志是告诉Sass其输入为标准输入。当它被传递时，不能传递任何输入文件。

$ echo "h1 {font-size: 40px}" | sass --stdin h1.css
$ echo "h1 {font-size: 40px}" | sass --stdin
h1 {
    font-size: 40px;
}

--stdin标志不能用于多对多模式。

# --indented

此标志告诉Sass将输入文件解析为缩进语法。如果以多对多模式使用，则所有输入文件都将被解析为缩进语法，use引入过来的文件则按其原来方式处理。相反--no-indented，可以用来强制将所有输入文件解析为SCSS。

--indented标记在输入为标准输入时非常有用，因此不能自动确定其语法。

$ echo -e 'h1\n  font-size: 40px' | sass --indented -
h1 {
    font-size: 40px;
}

# --load-path

这个选项（缩写为-I）为Sass添加了一个额外的加载路径来查找样式表。它可以被多次传递以提供多个加载路径。较早的加载路径将优先于较晚的加载路径。

$ sass --load-path=node_modules/bootstrap/dist/css style.scss style.css

# --style

此选项（缩写为-s）控制生成的CSS的输出样式。Dart Sass支持两种输出方式:

  • expanded 将每个选择器和声明各自一行。
  • compressed删除尽可能多的额外字符，并将整个样式表写入一行。
$ sass --style=expanded style.scss
h1 {
    font-size: 40px;
}

$ sass --style=compressed style.scss
h1{font-size:40px}

# --no-charset

兼容性 Dart Sass 1.19.0+

这个标志告诉Sass不要发出@charset声明或UTF-8字节顺序标记。默认情况下，如果--charset被传递，如果样式表包含任何非ASCII字符，Sass将插入@charset声明（在扩展输出模式下）或字节顺序标记（在压缩输出模式下）。

$ echo 'h1::before {content: "👭"}' | sass --no-charset
h1::before {
    content: "👭";
}

$ echo 'h1::before {content: "👭"}' | sass --charset
@charset "UTF-8";
h1::before {
    content: "👭";
}

# --error-css

兼容性 Dart Sass 1.20.0+

此标志告诉Sass在编译过程中出现错误时是否发出CSS文件。这个CSS文件在注释和body::before的内容属性中描述了错误，这样您就可以在浏览器中看到错误消息，而不需要切换回终端。

默认情况下，如果您正在编译磁盘上的至少一个文件（与标准输出相反），则会启用错误输出。您可以显式地传递--error-css来启用它，在编译到标准输出时也是如此，或者传递--no-error-css来在任何地方禁用它。当它被禁用时，--update标志和--watch标志将在出现错误时删除CSS文件。

$ sass --error-css style.scss style.css
/* Error: Incompatible units em and px.
 *   ,
 * 1 | $width: 15px + 2em;
 *   |         ^^^^^^^^^^
 *   '
 *   test.scss 1:9  root stylesheet */

body::before {
  font-family: "Source Code Pro", "SF Mono", Monaco, Inconsolata, "Fira Mono",
      "Droid Sans Mono", monospace, monospace;
  white-space: pre;
  display: block;
  padding: 1em;
  margin-bottom: 1em;
  border-bottom: 2px solid black;
  content: "Error: Incompatible units em and px.\a   \2577 \a 1 \2502  $width: 15px + 2em;\a   \2502          ^^^^^^^^^^\a   \2575 \a   test.scss 1:9  root stylesheet";         
}
Error: Incompatible units em and px.
  ╷
1 │ $width: 15px + 2em;
  │         ^^^^^^^^^^
  ╵
  test.scss 1:9  root stylesheet

# --update

兼容性 Dart Sass 1.4.0+

如果传递了--update标志，那么Sass将只编译那些最近修改了依赖关系的样式表，而不是生成相应的CSS文件。它还会在更新样式表时打印状态消息。

$ sass --update themes:public/css
Compiled themes/light.scss to public/css/light.css.

# Source Maps

兼容性 Dart Sass 1.3.0+

源映射是告诉浏览器或其他使用CSS的工具CSS如何与生成它的Sass文件相关联。它们使得在浏览器中查看甚至编辑Sass文件成为可能。参见在Chrome和Firefox中使用源映射的说明。

Dart Sass默认生成源映射。

# --no-source-map

如果传递了--no-source-map标志，Sass将不会生成任何源映射。它不能与其他源映射选项一起传递。

$ sass --no-source-map style.scss style.css

# --source-map-urls

此选项控制Sass生成的源映射如何链接回生成CSS。Dart Sass支持两种类型的URLs：

  • relative，默认值。使用从源映射文件位置到Sass源文件位置的相对URLs。
  • absolute,使用绝对file: URLs的Sass源文件。注意，绝对URLs只能在编译CSS的同一台计算机上工作。
​# Generates a URL like "../sass/style.scss".
$ sass --source-map-urls=relative sass/style.scss css/style.css

​# Generates a URL like "file:///home/style-wiz/sassy-app/sass/style.scss".
$ sass --source-map-urls=absolute sass/style.scss css/style.css

# --embed-sources

此标志告诉Sass将生成CSS的Sass文件的全部内容嵌入到源映射中。这可能会生成非常大的源映射，但它保证了无论如何使用CSS，源都可以在任何计算机上使用。

$ sass --embed-sources sass/style.scss css.style.css

# --embed-source-map

此标志告诉Sass将源映射文件的内容嵌入到生成的CSS中，而不是创建一个单独的文件并从CSS链接到它。

$ sass --embed-source-map sass/style.scss css.style.css

# 其它选项

# --watch

兼容性 Dart Sass 1.6.0+

此标志的作用类似于--update标志，但在第一轮编译完成后，Sass将保持打开状态，并在样式表或其依赖项发生更改时继续编译它们。

$ sass --watch themes:public/css
Compiled themes/light.scss to public/css/light.css.

​# Then when you edit themes/dark.scss...
Compiled themes/dark.scss to public/css/dark.css.

# --poll

兼容性 Dart Sass 1.8.0+

此标志（可能只与--watch一起传递）告诉Sass每隔一段时间手动检查源文件的更改，而不是依赖于操作系统在发生更改时通知它。如果您在操作系统的通知系统无法工作的远程驱动器上编辑Sass，这可能是必要的。

$ sass --watch --poll themes:public/css
Compiled themes/light.scss to public/css/light.css.

​# Then when you edit themes/dark.scss...
Compiled themes/dark.scss to public/css/dark.css.

# --top-on-error

兼容性 Dart Sass 1.8.0+

此标志告诉Sass在检测到错误时立即停止编译，而不是尝试编译其他可能不包含错误的Sass文件。它在多对多模式下非常有用。

$ sass --stop-on-error themes:public/css
Error: Expected expression.
   ╷
42 │ h1 {font-face: }
   │                ^
   ╵
  themes/light.scss 42:16  root stylesheet

# --interactive

兼容性 Dart Sass 1.5.0+

这个标志（缩写为-i）告诉Sass以交互模式运行，在交互模式中，您可以编写SassScript表达式并查看它们的结果。交互模式还支持变量和@use规则。

$ sass --interactive
>> 1px + 1in
97px
>> @use "sass:map"
>> $map: ("width": 100px, "height": 70px)
("width": 100px, "height": 70px)
>> map.get($map, "width")
100px

# --color

这个标志（缩写为-c）告诉Sass发出终端颜色，而相反的--no-color告诉它不发出颜色。默认情况下，如果它看起来像是在支持它们的终端上运行，它就会发出颜色。

$ sass --color style.scss style.css
Error: Incompatible units em and px.
  ╷
1 │ $width:  15px + 2em
  │          ^^^^^^^^^^
  ╵
  style.scss 1:9  root stylesheet

$ sass --no-color style.scss style.css
Error: Incompatible units em and px.
  ╷
1 │ $width: 15px + 2em
  │         ^^^^^^^^^^
  ╵
  style.scss 1:9  root stylesheet

# --no-unicode

兼容性 Dart Sass 1.17.0+

此标志告诉Sass仅将ASCII字符作为错误消息的一部分发送到终端。默认情况下，或者如果传递了--unicode， Sass将为这些消息发出非ASCII字符。此标记不影响CSS输出。

$ sass --no-unicode style.scss style.css
Error: Incompatible units em and px.
  ,
1 | $width: 15px + 2em;
  |         ^^^^^^^^^^
  '
  test.scss 1:9  root stylesheet

$ sass --unicode style.scss style.css
Error: Incompatible units em and px.
  ╷
1 │ $width: 15px + 2em;
  │         ^^^^^^^^^^
  ╵
  test.scss 1:9  root stylesheet

# --quiet

此标志（缩写为-q）告诉Sass在编译时不要发出任何警告。默认情况下，当使用过时的特性或遇到@warn规则时，Sass发出警告。它还使@debug规则静默。

$ sass --quiet style.scss style.css

# --help

此标记（缩写为-h）打印此文档的摘要。

$ sass --help
Compile Sass to CSS.

Usage: sass <input.scss> [output.css]
       sass <input.scss>:<output.css> <input/>:<output/>

...

# --version

此标志打印Sassd当前的版本。

$ sass --version
1.23.7
上次更新: 2019-11-22 11:19:47 AM
