# Ruby Sass命令行界面

Ruby Sass已经到了生命的尽头，现在完全没有得到维护。请尽快切换到Dart Sass或LibSass。

# 使用方法

Ruby Sass可执行文件可以在两种模式中调用。

# 一对一模式

sass [input.scss] [output.css]

一对一模式将单个输入文件（input.scss）编译为单个输出位置（output.css ）。如果没有传递输出位置，则将编译后的CSS打印到终端。如果没有传递输入或输出，则标准输入作为输入源并将其打印到终端。

如果输入文件的扩展名是.SCSS，则将其解析为SCSS；如果扩展名是.sass，则将其解析为缩进语法。如果它没有这两种扩展中的一种，或者它来自标准输入，则默认将其解析为缩进语法。这可以用--scss标志来控制。

# 多对多模式

sass [<input.css>:<output.css>] [<input/>:<output/>] [input.css] [input/]...

多对多模式将一个或多个输入文件编译为一个或多个输出文件。输入和输出用冒号分开。它还可以将一个目录中的所有Sass文件编译为另一个目录中具有相同名称的CSS文件。当任何参数包含冒号，或传递--update标志--watch标志时，将启用多对多模式。

如果传递的输入文件没有对应的输出文件，则将其编译为以输入文件命名的CSS文件，但扩展名将替换为.css。如果传递的输入目录没有对应的输出目录，则其中的所有Sass文件都将编译为同一目录中的CSS文件。

$ sass style.scss:style.css
      write style.css
      write style.css.map
$ sass light.scss:light.css dark.scss:dark.css
      write light.css
      write light.css.map
      write dark.css
      write dark.css.map
$ sass themes:public/css
      write public/css/light.css
      write public/css/light.css.map
      write public/css/dark.css
      write public/css/dark.css.map

在编译整个目录时，Sass将忽略名称以_开头的片段文件。您可使用片段文件来避免生成一堆不必要的输出文件。

多对多模式将只编译那些依赖项修改得比生成相应CSS文件更晚的样式表。它还会在更新样式表时打印状态消息。

# Ruby Sass命令行界面

# 配置选项

# 通用选项

# --load-path

这个选项（缩写为-I）为Sass添加了一个额外的加载路径来查找样式表。它可以被多次传递以提供多个加载路径。较早的加载路径将优先于较晚的加载路径。

$ sass --load-path=node_modules/bootstrap/dist/css style.scss style.css

如果设置了环境变量SASS_PATH，那么加载路径也将从该环境变量加载。该变量由;（Windows）或:（其它操作系统）作为分隔符。SASS_PATH上的加载路径优先于命令行上传递的加载路径。

$ SASS_PATH=node_modules/bootstrap/dist/css sass style.scss style.css

# --require

这个选项（缩写为-r）在运行Sass之前加载一个Ruby gem。它可用于将Ruby中定义的函数加载到您的Sass环境中。

$ sass --require=rails-sass-images style.scss style.css

# --compass

此标志加载Compass框架，并使其mixin和函数可用于Sass。

$ sass --compass style.scss style.css

# --style

此选项（缩写为-t）控制生成的CSS的输出样式。Ruby Sass支持四种输出风格：

  • nested，默认值，输出的CSS嵌套规则格式与Sass的缩进写法相同。
  • expanded，展开将每个选择器和声明写在自己的行上。
  • compact，将每个CSS规则放在单独的一行上。
  • compressed，压缩删除尽可能多的额外字符，并将整个样式表写入一行。
$ sass --style=nested
h1 {
  font-size: 40px; }
  h1 code {
    font-face: Roboto Mono; }

$ sass --style=expanded style.scss
h1 {
  font-size: 40px;
}
h1 code {
  font-face: Roboto Mono;
}

$ sass --style=compact style.scss
h1 { font-size: 40px; }
h1 code { font-face: Roboto Mono; }

$ sass --style=compressed style.scss
h1{font-size:40px}h1 code{font-face:Roboto Mono}

# --help

此标记（缩写为-h和-?）打印此文档的摘要。

$ sass --help
Usage: sass [options] [INPUT] [OUTPUT]

Description:
  Converts SCSS or Sass files to CSS.
...

# --version

此标志打印Sass的当前版本。

$ sass --version

# 监听和更新

这些选项会影响多对多模式。

# --watch

启用多对多模式，并使Sass在样式表或其依赖项发生更改时保持打开并继续编译样式表。

$ sass --watch themes:public/css
      write public/css/light.css
      write public/css/light.css.map

​# Then when you edit themes/dark.scss...
      write public/css/dark.css
      write public/css/dark.css.map

# --poll

此标志（可能只与--watch一起传递）告诉Sass每隔一段时间手动检查源文件的更改，而不是依赖于操作系统在发生更改时通知它。如果您在操作系统的通知系统无法工作的远程驱动器上编辑Sass，这可能是必要的。

$ sass --watch --poll themes:public/css
      write public/css/light.css
      write public/css/light.css.map

​# Then when you edit themes/dark.scss...
      write public/css/dark.css
      write public/css/dark.css.map

# --update

此标志启用多对多模式，即使所有参数都不是冒号分隔的对。

$ sass --update style.scss
      write style.css
      write style.css.map

# --force

此标志（缩写为-f）只能在多对多模式下传递。它使Sass文件总是被编译成CSS文件，而不是只在源文件比输出更新得快的时候才编译。

--force标志不能与--watch标志一起使用。

$ sass --force style.scss:style.css
      write style.css
      write style.css.map

# --stop-on-error

此标志只能在多对多模式下传递。它告诉Sass在检测到错误时立即停止编译，而不是试图编译其它可能不包含错误的Sass文件。

$ sass --stop-on-error themes:public/css
Error: Invalid CSS after "h1 {font-size: ": expected expression (e.g. 1px, bold), was "}"
        on line 1 of test.scss
  Use --trace for backtrace.

# 输入与输出

这些选项控制Sass如何加载输入文件以及如何生成输出文件。

# --scss

此标志告诉Sass将标准输入解析为SCSS。

$ echo "h1 {font-size: 40px}" | sass --scss
h1 {
  font-size: 40px;
}

# --sourcemap

此选项控制Sass如何生成源映射，源映射是告诉浏览器或其他使用CSS的工具CSS如何与生成它的Sass文件相关联。它们使得在浏览器中查看甚至编辑Sass文件成为可能。参见在Chrome和Firefox中使用源映射的说明。它有四个可能的值：

  • auto，默认值，使用相对URLs将源映射链接到Sass样式表（如果可能的话），否则使用绝对file: URLs。
  • file，使用绝对file:URLs链接源映射到Sass样式表。
  • inline，源映射中直接包含的Sass样式表的文本。
  • none，本不生成源映射。
​# Generates a URL like "../sass/style.scss".
$ sass --sourcemap=auto sass/style.scss css/style.css

​# Generates a URL like "file:///home/style-wiz/sassy-app/sass/style.scss".
$ sass --sourcemap=file sass/style.scss css/style.css

​# Includes the full text of sass/style.scss in the source map.
$ sass --sourcemap=inline sass/style.scss css/style.css

​# Doesn't generate a source map.
$ sass --sourcemap=none sass/style.scss css/style.css

# --stdin

此标志（缩写为-s）告诉Sass其输入为标准输入。当它被传递时，不能传递任何输入文件。

$ echo -e 'h1\n  font-size: 40px' | sass --stdin
h1 {
  font-size: 40px;
}

--stdin标志不能用于多对多模式。

# --default-encoding

缩写为-E，控制Sass在加载没有显式指定字符编码的源文件时，使用默认字符编码。它默认为操作系统的默认编码。

$ sass --default-encoding=Shift-JIS style.scss style.css

# --unix-newlines

此标志告诉Sass生成的输出文件的行与U+000A换行符分隔，而不是操作系统的默认值（在Windows上，这是U+000D回车后跟U+000A换行符）。在默认为Unix样式的换行的系统上总是正确的。

$ sass --unix-newlines style.scss style.css

# --debug-info

此标志（缩写为-g）导致Sass发出伪@media查询，该查询指示每个样式规则在源样式表中定义的位置。

$ sass --debug-info style.scss
@media -sass-debug-info{filename{font-family:file\:\/\/\/home\/style-wiz\/sassy-app\/style\.scss}line{font-family:\000031}}
h1 {
  font-size: 40px; }

::: waning 此标志仅用于向后兼容。现在推荐使用源映射将CSS映射回生成它的Sass。 :::

# --line-comments

此标志（也可以是--line-numbers，缩写为-l）使得Sass对每个样式规则发出注释，这些注释指示源样式表中每个样式规则的定义位置。

$ sass --line-numbers style.scss
/* line 1, style.scss */
h1 {
  font-size: 40px; }

# 其它选项

# --interactive

这个标志（缩写为-i）告诉Sass以交互模式运行，在交互模式中，您可以编写SassScript表达式并查看它们的结果。交互模式还支持变量。

$ sass --interactive
>> 1px + 1in
97px
>> $map: ("width": 100px, "height": 70px)
("width": 100px, "height": 70px)
>> map-get($map, "width")
100px

# --check

这个标志（缩写为-c）告诉Sass在不执行该文件的情况下验证其输入文件的语法是否有效。如果语法有效，它将以状态0退出。它不能在多对多模式下使用。

$ sass --check style.scss

# --precision

此选项告诉Sass在发出十进制数时要使用多少位精度数字。

$ echo -e 'h1\n  font-size: (100px / 3)' | sass --precision=20
h1 {
  font-size: 33.333333333333336px; }

# --cache-location

此选项告诉Sass将其解析文件的缓存存储在何处，以便在未来的调用中运行得更快，默认是.sass-cache。

$ sass --cache-location=/tmp/sass-cache style.scss style.css

# --no-cache

此标志（缩写为-C）告诉Sass不缓存已解析的文件。

$ sass --no-cache style.scss style.css

# --quiet

此标志（缩写为-q）告诉Sass在编译时不要发出任何警告。默认情况下，当使用过时的特性或遇到@warn规则时，Sass发出警告。它还使@debug规则失效。

$ sass --quiet style.scss style.css
